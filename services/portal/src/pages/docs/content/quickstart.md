# Quickstart

This walks you through one full loop: ingest content, build a knowledge graph, ask questions, and browse the auto-distilled Wiki. It assumes the platform is [installed](installation.md) and [activated](activation.md), with the API at `http://localhost:4000`.

## The end-to-end flow

```
KEX (extract) → FUSE (merge + resolve) → Talk-to-Graph (RAG) → Wiki (browse)
```

## 1. Ingest with KEX

Add a document or raw text. In the dashboard, use **Upload / Import**, or call the API directly.

```bash
curl -X POST http://localhost:4000/kex/extract \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Ada Lovelace worked with Charles Babbage on the Analytical Engine in 1843.",
    "name": "lovelace-note"
  }'
```

KEX extracts entities and relations from the source and returns an extraction you can feed into FUSE.

## 2. Merge and resolve with FUSE

FUSE merges your extractions and **resolves entities** (deduplicating and linking them) into a **compilation** — a knowledge graph. Trigger it from the dashboard, or:

```bash
curl -X POST http://localhost:4000/fuse/compile \
  -H "Content-Type: application/json" \
  -d '{
    "compilation": "my-first-graph",
    "extractions": ["lovelace-note"]
  }'
```

The result is a single clean graph with duplicates collapsed and entities linked.

## 3. Ask questions in Talk-to-Graph

Talk-to-Graph is the **RAG** interface over your compilation. Ask in natural language from the dashboard, or via the API:

```bash
curl -X POST http://localhost:4000/rag/query \
  -H "Content-Type: application/json" \
  -d '{
    "compilation": "my-first-graph",
    "question": "Who did Ada Lovelace work with?"
  }'
```

Answers are grounded in the graph and generated by your local model — nothing leaves the machine.

## 4. Browse the Wiki

GCTRL auto-distills a human-readable **Wiki** over your compilation. Open the **Wiki** view in the dashboard to browse entities and their relationships as linked pages — a curated knowledge surface over the same graph that powers RAG.
